UODO RAG API

⚡ FastAPI 0.115 🔍 OpenSearch 2.18 🤖 Ollama / Groq 📄 ~560 decyzji UODO

REST API systemu RAG (Retrieval-Augmented Generation) do przeszukiwania decyzji Prezesa UODO, ustawy o ochronie danych osobowych oraz RODO. Łączy wyszukiwanie hybrydowe (BM25 + kNN) z gramatycznym grafem cytowań i streamingiem odpowiedzi LLM.

Szybki start

Base URL http://localhost:8000

Minimalne zapytanie wyszukujące:

# Wyszukiwanie decyzji o danych genetycznych
curl -s -X POST http://localhost:8000/api/search \
  -H "Content-Type: application/json" \
  -d '{"query": "dane genetyczne"}' | python3 -m json.tool

Streaming odpowiedzi AI (SSE):

# Najpierw pobierz docs przez /api/search, potem wyślij do stream
curl -N -X POST http://localhost:8000/api/answer/stream \
  -H "Content-Type: application/json" \
  -d '{"query":"dane genetyczne","docs":[...],"provider":"Ollama","model":"mistral-large-3:675b-cloud"}'
Swagger UI dostępny pod /docs — pozwala testować wszystkie endpointy interaktywnie.

Widget frontend

Gotowy widget React do osadzenia na dowolnej stronie:

Metoda 1 — Web Component (automatyczna inicjalizacja)
<script src="https://rag.uodo.gov.pl/widget.iife.js"></script>
<uodo-rag-widget api-url="https://rag.uodo.gov.pl"></uodo-rag-widget>
Metoda 2 — ręczna inicjalizacja
<div id="uodo-rag"></div>
<script src="widget.iife.js"></script>
<script>
  UodoRag.mount('#uodo-rag', {
    useLLM:          true,
    onDocumentClick: (doc) => console.log(doc.signature),
  });
</script>

Wyszukiwanie

POST /api/search Wyszukiwanie hybrydowe ▼

Wyszukiwanie hybrydowe łączące BM25 (dopasowanie słów kluczowych), kNN (wyszukiwanie semantyczne) i graf cytowań. Decyzje UODO, artykuły u.o.d.o. i RODO w jednym wywołaniu.

Request Body
PoleTypOpis
query *stringZapytanie użytkownika
search_querystringWzbogacone zapytanie (z /api/decompose). Jeśli podane, używane do embeddingów zamiast query.
filters.statusstring"prawomocna" | "nieprawomocna"
filters.year_fromintegerRok wydania od (np. 2022)
filters.year_tointegerRok wydania do
filters.term_sectorstring[]Sektor: ["Zdrowie", "Zatrudnienie"]
filters.term_decision_typestring[]Rodzaj decyzji: ["nakaz", "upomnienie"]
use_graphbooleanRozszerz wyniki o powiązane decyzje z grafu cytowań (domyślnie: true)
top_kintegerMaksymalna liczba wyników (1–50, domyślnie: 8)
Przykład
{
  "query":   "dane genetyczne monitoring pracowników",
  "filters": { "term_sector": ["Zatrudnienie"], "year_from": 2022 },
  "top_k":   20
}
Response
PoleOpis
docs[]Lista dokumentów (decyzje, artykuły u.o.d.o., RODO)
docs[].doc_typeuodo_decision | legal_act_article | gdpr_article | gdpr_recital
docs[].signatureSygnatura decyzji (np. DKN.5131.5.2024)
docs[].section_titleTytuł najlepiej pasującej sekcji
docs[].scoreWynik dopasowania
docs[].graph_relationRelacja z grafu cytowań (jeśli dotyczy)
tags[]Tagi LLM dopasowane do zapytania
search_timeCzas wyszukiwania w sekundach
POST /api/answer/stream Streaming odpowiedzi LLM (SSE) ▼

Generuje odpowiedź AI na podstawie zapytania i dokumentów z /api/search. Odpowiedź streamowana przez Server-Sent Events — kolejne tokeny przychodzą na bieżąco bez czekania na całość.

Request Body
PoleTypOpis
query *stringOryginalne zapytanie
docs *Document[]Dokumenty z /api/search (zwykle pierwsze 8)
providerstring"Ollama" | "Groq"
modelstringNazwa modelu (np. "mistral-large-3:675b-cloud")
Format zdarzeń SSE
# Token odpowiedzi
data: {"type": "token", "content": "Na podstawie"}

# Koniec streamu
data: {"type": "done"}

# Błąd
data: {"type": "error", "message": "..."}
Przykład JavaScript
const resp = await fetch('/api/answer/stream', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ query, docs, provider: 'Ollama', model }),
});
const reader = resp.body.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  // parsuj zdarzenia SSE z decoder.decode(value)
}
Nginx wymaga proxy_buffering off dla poprawnego działania SSE.
POST /api/decompose Dekompozycja zapytania (Reasoning Step) ▼

LLM analizuje zapytanie użytkownika i generuje ustrukturyzowane parametry wyszukiwania: słowa kluczowe, wskazówki artykułów, zakres lat. Wywołaj przed /api/search dla złożonych zapytań (powyżej 3 słów).

Request Body
PoleOpis
query *Zapytanie do dekompozycji
providerProvider LLM
modelModel LLM
Response
PoleOpis
enriched_queryRozszerzone zapytanie — użyj jako search_query w /api/search
search_keywords[]Synonimy i pojęcia prawne (max 5)
gdpr_articles_hint[]Sugerowane artykuły RODO
year_from_hintSugerowany rok od (lub null)
reasoningUzasadnienie dekompozycji
GET /api/suggest Autouzupełnianie ▼

Zwraca tagi i sygnatury pasujące do prefiksu zapytania. Minimum 2 znaki. Używane przez pole wyszukiwania w widgecie.

Query Parameters
ParametrOpis
q *Prefiks (min. 2 znaki)
limitMaksymalna liczba wyników (domyślnie: 8)
Przykład
GET /api/suggest?q=dane+gene&limit=6

{
  "tags":       ["dane genetyczne", "dane biometryczne"],
  "signatures": ["DKN.5131.32.2023", "DKN.5131.9.2024"]
}

Dokumenty

GET /api/document Pełna treść decyzji (wszystkie sekcje) ▼

Pobiera pełną decyzję UODO — wszystkie sekcje w kolejności (Sentencja, Stan faktyczny, sekcje uzasadnienia) wraz z referencjami (URN) do aktów prawnych, orzeczeń sądów i innych decyzji UODO.

Query Parameters
ParametrOpis
signature *Sygnatura decyzji, np. DKN.5131.5.2024
Response (wybrane pola)
PoleOpis
sections[]Lista sekcji: {section_title, section_id, chunk_index, content_text}
refs_full[]Referencje z URN: {urn, signature, category, display}
related_acts[]Akty prawne PL (sygnatury)
related_uodo_rulings[]Cytowane decyzje UODO
Przykład
GET /api/document?signature=DKN.5131.5.2024
GET /api/signature/{signature} Metadane decyzji (chunk 0) ▼

Pobiera metadane i sentencję decyzji UODO (chunk 0). Szybsze niż /api/document gdy potrzebne są tylko metadane.

Uwaga: sygnatura w ścieżce URL musi być zakodowana — DKN.5131.5.2024 → DKN.5131.5.2024 (kropki nie wymagają kodowania).

Metadane

GET /api/tags Wszystkie tagi (autocomplete filtrów) ▼

Zwraca wszystkie unikalne tagi z bazy decyzji, posortowane alfabetycznie. Cache TTL: 1 godzina. 

GET /api/tags
→ ["administracyjna kara pieniężna", "dane biometryczne", ...]
GET /api/taxonomy Opcje filtrów taksonomii ▼

Zwraca dostępne wartości dla każdej kategorii filtrów. Używane przez sidebar filtrów w widgecie. 

GET /api/taxonomy
→ {
  "term_decision_type":      ["nakaz", "upomnienie", ...],
  "term_sector":             ["Zdrowie", "Zatrudnienie", ...],
  "term_corrective_measure": [...],
  "term_violation_type":     [...]
}

Admin

POST /api/admin/update Delta-aktualizacja decyzji ▼

Wyzwala pobieranie i indeksowanie nowych decyzji UODO. Automatycznie wykrywa ostatnią zaindeksowaną datę i pobiera tylko nowsze decyzje (delta update).

Endpoint wymaga parametru ?admin_key=... zgodnego z ADMIN_KEY w pliku .env. Bez klucza dostęp jest otwarty (ustaw klucz w produkcji!).
Query Parameters
ParametrOpis
admin_keyKlucz dostępu (z .env)
Request Body
PoleOpis
date_fromPobierz od daty YYYY-MM-DD (domyślnie: auto)
dry_runTylko sprawdź, nie indeksuj (domyślnie: false)
force_allPrzeindeksuj wszystkie decyzje (domyślnie: false)
batch_sizeBatch embeddingów (domyślnie: 8)
GET /api/admin/update/status Status ostatniej aktualizacji ▼ 
GET /api/admin/update/status
→ {
  "last_update":       "2026-04-03T06:12:44",
  "last_indexed_date": "2026-04-01",
  "total_indexed":     564
}
